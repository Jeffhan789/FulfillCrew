# 云端多智能体智能电商系统：中文项目介绍

这个项目是一个面向 GitHub 展示、面试讲解和本科生学习参考的 AI + Multi-Agent 电商系统。

它的出发点很明确：不要只做一个普通的商品列表和购物篮，而是把一个简单电商原型升级成更完整的智能订单系统。用户在前端下单后，后端会触发多个 Agent 协作完成风险检测、库存检查、仓库竞价、需求预测和订单状态返回。新版前端还加入了课程映射 dashboard，让读者能直接看到 COMP315、COMP310 和 ELEC320 分别如何落在系统中。

## 项目定位

项目英文名是 `Cloud-Based Multi-Agent E-Commerce Intelligence System`，中文名是“云端多智能体智能电商系统”。

它不是一个为了复杂而复杂的项目，而是一个适合展示工程思维的项目。项目把数据处理、React 前端、FastAPI 后端、多智能体系统和神经网络预测接口放进同一个业务流程里，让每个技术点都有真实位置。

## 为什么适合面试展示

面试时，这个项目可以用一条 checkout 流程讲清楚：

1. 商品数据先经过 JavaScript cleaning pipeline 清洗。
2. React 前端展示商品、支持搜索、排序、库存筛选和购物篮。
3. 用户提交 checkout 后，请求进入 FastAPI 后端。
4. Order Agent 接收订单。
5. Fraud Detection Agent 输出风险分数。
6. Inventory Agent 检查库存。
7. Coordinator Agent 向多个 Warehouse Agent 请求 bid。
8. Warehouse Agent 根据库存、距离、负载和处理速度提交 bid。
9. Coordinator Agent 选择最优仓库，并展示每个仓库 bid 的原因。
10. Demand Prediction Agent 输出未来 7 天需求预测。
11. 前端显示订单状态、风险分数、仓库选择、模型证据、课程 trace 和 Agent decision log。

这条链路能证明你不只是会写页面，还能理解系统如何分层、模块如何协作、业务流程如何落地。

## 为什么适合本科生学习

很多本科项目容易停留在 CRUD 或静态网页层面。这个项目的价值在于，它提供了一个可以继续扩展的骨架：

- 想练前端：可以优化 React UI、加入图表和订单详情页。
- 想练后端：可以扩展 FastAPI 路由、订单状态机、数据库和认证。
- 想练 AI：可以把当前轻量预测模块替换成真正训练好的 MLP、SVM 或 PyTorch 模型。
- 想练 Multi-Agent：可以扩展 Contract Net Protocol、Agent message log 和协作策略。
- 想练部署：可以加入 Docker Compose、PostgreSQL、CI/CD 和云服务器部署。

换句话说，它既能作为一个可运行的第一版，也能作为 final-year project 或 portfolio project 的长期升级底座。

## 当前版本包含什么

当前 MVP 已经包含以下部分：

- `data_cleaning/`：商品数据清洗脚本与示例数据。
- `frontend/`：React 电商前端，包含商品展示、搜索、排序、basket 和 checkout。
- `backend/`：FastAPI 后端，包含产品 API、订单 API 和 Agent 编排。
- `backend/agents/`：Order、Inventory、Coordinator、Warehouse、Demand Prediction、Fraud Detection 等 Agent。
- `ml_models/`：需求预测、欺诈检测和商品分类模型接口。
- `docs/`：系统设计、Agent 设计、模型说明、部署说明和中文解释文档。
- `tests/`：基础 smoke tests，用于验证数据清洗和 Agent 流程。

## 本次功能升级亮点

新版项目更贴近课程内容，也更适合 GitHub 展示：

- 增加课程映射接口，让 COMP315、COMP310、ELEC320 的项目对应关系可以被前端直接展示。
- Warehouse Agent 的 bid 不再只是一个数字，而是带有 workload、stock、distance、processing speed 和解释原因。
- 订单结果增加 course trace，帮助面试官快速理解每一层技术来自哪门课。
- 模型部分增加 model evaluation 摘要，用 training mode 和 online mode 解释 ELEC320 的神经网络思路。
- 前端增加 Course Intelligence Dashboard，让本科生不用读后端代码也能理解项目结构。

## 后续可以怎么升级

下一步最值得做的升级包括：

1. 加入 SQLite 或 PostgreSQL，保存商品、订单、库存和 Agent logs。
2. 用真实训练数据替换当前轻量预测模型。
3. 在前端加入 dashboard，展示 warehouse bids、risk score 和 demand prediction。
4. 增加截图、demo GIF 和在线部署链接，让 GitHub 首页更吸引人。
5. 增加英文/中文项目讲解文章，帮助其他本科生理解如何从 coursework 升级到 portfolio。

## 一句话总结

这是一个把 coursework、AI、Multi-Agent 和云端工程整合起来的展示型项目。它适合面试，也适合本科生 fork 学习。如果这个项目能帮助你规划自己的 portfolio 或 final-year project，欢迎 star 支持。
