# FulfillCrew: Chinese Project Introduction

> **Cloud-Based Multi-Agent E-Commerce Intelligence System**
>
> 这是一个面向 GitHub 展示、面试讲解和本科生学习参考的 AI + Multi-Agent 电商系统。

## Project Positioning

The English name is `Cloud-Based Multi-Agent E-Commerce Intelligence System`. The Chinese name is "智仓通".

It is not a project built for complexity's sake, but a project suitable for demonstrating engineering thinking. The system integrates data processing, React frontend, FastAPI backend, multi-agent systems and neural network prediction interfaces into a single business flow, giving each technology a real and meaningful place.

## Why It Is Suitable for Interview Demonstrations

During an interview, this project can explain a clear checkout flow:

1. Product data first passes through the JavaScript cleaning pipeline.
2. The React frontend displays products, supports search, sorting, stock filtering and basket.
3. After the user submits checkout, the request enters the FastAPI backend.
4. Order Agent receives the order.
5. Fraud Detection Agent outputs a risk score.
6. Inventory Agent checks stock.
7. Coordinator Agent requests bids from multiple Warehouse Agents.
8. Warehouse Agent submits bids based on stock, distance, workload and processing speed.
9. Coordinator Agent selects the optimal warehouse and displays the reason for each bid.
10. Demand Prediction Agent outputs the next 7-day demand forecast.
11. The frontend displays order status, risk score, warehouse selection, model evidence, course trace and agent decision log.

This chain proves that you don't just know how to write pages, but also understand how systems are layered, how modules collaborate and how business processes are implemented.

## Why It Is Suitable for Undergraduate Learning

Many undergraduate projects stagnate at the CRUD or static webpage level. The value of this project is that it provides a skeleton that can continue to grow:

- Want to practice frontend? Optimise the React UI, add charts and order detail pages.
- Want to practice backend? Extend FastAPI routes, add an order state machine, database and authentication.
- Want to practice AI? Replace the current lightweight prediction modules with real trained MLP, SVM or PyTorch models.
- Want to practice Multi-Agent? Extend the Contract Net Protocol, add agent message logs and collaboration strategies.
- Want to practice deployment? Add Docker Compose, PostgreSQL, CI/CD and cloud server deployment.

In other words, it is both a runnable first version and a long-term upgrade base for final-year projects or portfolio projects.

## What the Current Version Contains

The current MVP already includes the following:

- `data_cleaning/`: Product data cleaning scripts and sample data.
- `frontend/`: React e-commerce frontend, including product display, search, sorting, basket and checkout.
- `backend/`: FastAPI backend, including product API, order API and agent orchestration.
- `backend/agents/`: Order, Inventory, Coordinator, Warehouse, Demand Prediction, Fraud Detection and other agents.
- `ml_models/`: Demand prediction, fraud detection and product category classifier interfaces.
- `docs/`: System design, agent design, model notes, deployment notes and Chinese explanation documents.
- `tests/`: Basic smoke tests for verifying data cleaning and agent flows.

## Highlights of This Upgrade

The new version is closer to course content and more suitable for GitHub showcase:

- Added course mapping interface so that COMP315, COMP310 and ELEC320 project correspondences can be directly displayed on the frontend.
- Warehouse Agent bids are no longer just a number, but include workload, stock, distance, processing speed and an explanatory reason.
- Order results include course trace to help interviewers quickly understand which technical layer comes from which course.
- Model section adds model evaluation summaries, using training mode and online mode to explain ELEC320 neural network thinking.
- Frontend adds a Course Intelligence Dashboard so undergraduates can understand the project structure without reading backend code.

## Next Upgrade Suggestions

The most worthwhile next upgrades include:

1. Add SQLite or PostgreSQL to persist products, orders, inventory and agent logs.
2. Replace the current lightweight prediction models with real training data.
3. Add a dashboard to the frontend to display warehouse bids, risk score and demand prediction.
4. Add screenshots, demo GIFs and an online deployment link to make the GitHub homepage more attractive.
5. Add English/Chinese project explanation articles to help other undergraduates understand how to upgrade from coursework to portfolio.

## One-Sentence Summary

This is a showcase project that integrates coursework, AI, Multi-Agent and cloud engineering. It is suitable for interviews and for undergraduates to fork and learn. If this project helps you plan your own portfolio or final-year project, welcome to star and support.

# 智仓通：中文项目介绍

> **智仓通（FulfillCrew）**
>
> 这是一个面向 GitHub 展示、面试讲解和本科生学习参考的 AI + Multi-Agent 电商系统。

## 项目定位

项目英文名是 `Cloud-Based Multi-Agent E-Commerce Intelligence System`，中文名是"智仓通"。

它不是一个为了复杂而复杂的项目，而是一个适合展示工程思维的项目。项目把数据处理、React 前端、FastAPI 后端、多智能体系统和神经网络预测接口放进同一个业务流程里，让每个技术点都有真实位置。

## 为什么适合面试展示

面试时，这个项目可以用一条 checkout 流程讲清楚：

1. 商品数据先经过 JavaScript cleaning pipeline 清洗。
2. React 前端展示商品、支持搜索、排序、库存筛选和购物篮。
3. 用户提交 checkout 后，请求进入 FastAPI 后端。
4. Order Agent 接收订单。
5. Fraud Detection Agent 输出风险分数。
6. Inventory Agent 检查库存。
7. Coordinator Agent 向多个 Warehouse Agent 请求 bid。
8. Warehouse Agent 根据库存、距离、负载和处理速度提交 bid。
9. Coordinator Agent 选择最优仓库，并展示每个仓库 bid 的原因。
10. Demand Prediction Agent 输出未来 7 天需求预测。
11. 前端显示订单状态、风险分数、仓库选择、模型证据、课程 trace 和 Agent decision log。

这条链路能证明你不只是会写页面，还能理解系统如何分层、模块如何协作、业务流程如何落地。

## 为什么适合本科生学习

很多本科项目容易停留在 CRUD 或静态网页层面。这个项目的价值在于，它提供了一个可以继续扩展的骨架：

- 想练前端：可以优化 React UI、加入图表和订单详情页。
- 想练后端：可以扩展 FastAPI 路由、订单状态机、数据库和认证。
- 想练 AI：可以把当前轻量预测模块替换成真正训练好的 MLP、SVM 或 PyTorch 模型。
- 想练 Multi-Agent：可以扩展 Contract Net Protocol、Agent message log 和协作策略。
- 想练部署：可以扩展 Docker Compose 配置、添加 PostgreSQL、CI/CD 和云服务器部署。

换句话说，它既能作为一个可运行的第一版，也能作为 final-year project 或 portfolio project 的长期升级底座。

## 当前版本包含什么

当前 MVP 已经包含以下部分：

- `data_cleaning/`：商品数据清洗脚本与示例数据。
- `frontend/`：React 电商前端，包含商品展示、搜索、排序、basket 和 checkout。
- `backend/`：FastAPI 后端，包含产品 API、订单 API 和 Agent 编排。
- `backend/agents/`：Order、Inventory、Coordinator、Warehouse、Demand Prediction、Fraud Detection 等 Agent。
- `ml_models/`：需求预测、欺诈检测和商品分类模型接口。
- `docs/`：系统设计、Agent 设计、模型说明、部署说明和中文解释文档。
- `tests/`：基础 smoke tests，用于验证数据清洗和 Agent 流程。

## 本次功能升级亮点

新版项目更贴近课程内容，也更适合 GitHub 展示：

- 增加课程映射接口，让 COMP315、COMP310、ELEC320 的项目对应关系可以被前端直接展示。
- Warehouse Agent 的 bid 不再只是一个数字，而是带有 workload、stock、distance、processing speed 和解释原因。
- 订单结果增加 course trace，帮助面试官快速理解每一层技术来自哪门课。
- 模型部分增加 model evaluation 摘要，用 training mode 和 online mode 解释 ELEC320 的神经网络思路。
- 前端增加 Course Intelligence Dashboard，让本科生不用读后端代码也能理解项目结构。

## 后续可以怎么升级

下一步最值得做的升级包括：

1. 加入 SQLite 或 PostgreSQL，保存商品、订单、库存和 Agent logs。
2. 用真实训练数据替换当前轻量预测模型。
3. 在前端加入 dashboard，展示 warehouse bids、risk score 和 demand prediction。
4. 增加截图、demo GIF 和在线部署链接，让 GitHub 首页更吸引人。
5. 增加英文/中文项目讲解文章，帮助其他本科生理解如何从 coursework 升级到 portfolio。

## 一句话总结

这是一个把 coursework、AI、Multi-Agent 和云端工程整合起来的展示型项目。它适合面试，也适合本科生 fork 学习。如果这个项目能帮助你规划自己的 portfolio 或 final-year project，欢迎 star 支持。
